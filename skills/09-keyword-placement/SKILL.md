# Keyword Placement — Natural Integration

## Objective
Integrate target keywords naturally throughout the visa guide at optimal density and strategic positions. Keywords must read naturally for humans while signaling relevance to search engines and AI overviews.

## Keyword Hierarchy

```
Primary Keyword (1-2) ────── Main target, used structurally
    │
Secondary Keywords (5-8) ─── Supporting keywords in H2s and body
    │
Long-tail / Question Kw (10-15) ── Natural inclusion in FAQ and body
    │
LSI / Related Keywords ────── Contextual synonyms and related terms
```

### Example Hierarchy for "Canada PR Visa for Indians"
```markdown
Primary: "Canada PR visa for Indians"
Secondary: "Canada permanent residence", "Express Entry for Indians", 
           "Canada PR eligibility", "Canada PR fees INR"
Long-tail: "How to get Canada PR from India", "CRS score for Indian applicants",
           "Canada PR processing time 2026"
LSI: "immigrate to Canada", "Canada work permit", "Canadian experience class",
      "PNP nomination", "ITA Canada"
```

## Strategic Keyword Placement Map

### Where Keywords MUST Appear

| Location | Keyword Type | Frequency |
|----------|-------------|-----------|
| **H1 (Page Title)** | Primary | Exactly once |
| **Meta Title** | Primary | Once |
| **Meta Description** | Primary + 1 secondary | Naturally |
| **URL Slug** | Primary | Once (hyphenated) |
| **First 100 words** | Primary | Once (naturally) |
| **H2 Subheadings** | Primary + secondary | 2-3 H2s |
| **Alt text (images)** | Primary / secondary | Each image |
| **FAQ questions** | Long-tail / question | 5+ FAQs |
| **Conclusion** | Primary | Once |
| **Internal link anchor** | Primary / secondary | 2-3 links |

### Where Keywords SHOULD Appear
| Location | Keyword Type | Notes |
|----------|-------------|-------|
| Body paragraphs (every 300 words) | Secondary / LSI | |
| Table headers / content | Secondary | Natural in table context |
| Bold / highlighted text | Primary | Once in intro |
| Image file names | Primary | Name files with keywords |
| Schema markup | Primary | FAQ and HowTo schema |

## Keyword Density Guidelines

| Content Length | Primary Keyword Max | Total Keyword Instances |
|---------------|-------------------|----------------------|
| 1000 words | 3-4 times | 8-12 total |
| 1500 words | 4-6 times | 12-18 total |
| 2000 words | 5-7 times | 15-22 total |
| 2500 words | 6-8 times | 18-26 total |
| 3000+ words | 8-10 times | 22-30 total |

**Rule of thumb:** If you remove the keyword and the sentence still makes sense, density is probably fine. If removal breaks the sentence, you're forcing it.

## Natural Keyword Integration Techniques

### Technique 1: The "Sandwich" Method
```
[Context sentence] + [Keyword sentence] + [Context sentence]

Example:
"The Canadian immigration system offers multiple pathways for skilled workers.
The Canada PR visa for Indians is particularly popular due to its straightforward
Express Entry system. You can apply from India without a job offer."
```

### Technique 2: Question-Answer Integration
```
H2: "How to Get Canada PR from India?"
→ Naturally uses long-tail keyword as heading
→ Answer body includes related keywords
→ AI Overview friendly format
```

### Technique 3: Definition Integration (Best for AI Overviews)
```
"What is Canada PR Visa for Indians?
The Canada PR visa for Indians is a permanent residence permit that allows
Indian citizens to live and work in Canada indefinitely. It's the most sought-after
immigration pathway for skilled Indian professionals."
```

### Technique 4: Table Integration
```markdown
| Visa Type | Processing for Indians | Fee (INR) | Success Rate |
|-----------|----------------------|-----------|--------------|
| Canada PR | 6-8 months | ₹XX,XXX | High |
| Canada Student | 4-6 months | ₹XX,XXX | Very High |
```

### Technique 5: FAQ Integration (Best for AI Overviews)
```
Q: What is the CRS score required for Canada PR from India?
A: The minimum CRS score for Canada PR has been... [naturally includes keywords]
```

## LSI Keyword Integration

LSI keywords show Google your content is comprehensive. Map them to sections:

| Section | LSI Keywords to Include |
|---------|----------------------|
| Introduction | "move abroad", "immigration", "work overseas", "settle in [country]" |
| Eligibility | "points calculator", "qualification assessment", "language proficiency" |
| Process | "application form", "submission", "appointment", "biometrics" |
| Fees | "processing fee", "service charge", "refund policy", "payment methods India" |
| Tips | "avoid rejection", "common mistakes", "interview preparation" |

## What NOT to Do

```
✗ Keyword stuffing: "Canada PR visa for Indians is the best Canada PR visa for
  Indians because Canada PR visa for Indians offers..."

✗ Forced exact match: "If you want a Canada PR visa from India, then this
  Canada PR visa for Indians guide..."

✗ Invisible keywords: White text, hidden divs — Google penalizes this

✗ Over-optimized headings: Every H2 starts with the same keyword

✗ Irrelevant keywords: Adding keywords that don't match content
```

## Step-by-Step Placement Process

1. **Write naturally first** — Generate content without worrying about keywords
2. **Identify placements** — Mark where primary keyword fits naturally
3. **Insert primary keyword** — H1, first 100 words, 1 H2, conclusion
4. **Insert secondary keywords** — Remaining H2s, body paragraphs
5. **Insert long-tail keywords** — FAQ section, natural body inclusion
6. **Add LSI keywords** — Throughout body for semantic relevance
7. **Check density** — Count occurrences, ensure within limits
8. **Read aloud test** — Does it sound natural? If awkward, revise
9. **AI overview test** — Would Google's AI extract this as a good answer?
